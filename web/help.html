Help & Syntax Guide

This guide reflects what LocalDocsMD actually does today, including markdown syntax, advanced blocks, shortcuts, and storage/auth behavior.

Server-backed app LocalDocsMD runs on your local server process and stores data on that server instance.
Account required You sign in with a user account; permissions are enforced via roles/workspace access.
Data persists locally Documents/projects are persisted by the app backend (not only in browser memory).

Supported callout tags include note, tip, warning, danger, info, and others.

Callout Types

Callouts are markdown blockquotes with a type marker: > [!TYPE] Title.

> [!TIP] Performance
> Keep large tables under collapsible headings.

> [!QUESTION] Why this design?
> It minimizes rerender cost while preserving readability.

> [!SUCCESS] Migration complete
> All workspaces now use role-based access.

Common types: note, tip, warning, danger, question, success.

Two supported forms: ++Ctrl+Shift+P++ and [[Cmd+S]].

Use ++Ctrl+S++ to force save.
Use ++Ctrl+Shift+P++ for command palette.
On macOS: [[Cmd+K]] then [[Cmd+S]].

Sharing, Access, and Data Accuracy

Document URLs are served by the backend and protected by authentication/authorization rules. Access depends on your account and role assignments.

LocalDocsMD is not a static browser-only app. It requires the running server process and uses backend persistence for documents/projects/workspaces.
